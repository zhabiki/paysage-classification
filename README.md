# Легендарная команда Ж А Б И К И представляет
# Классификация пейзажей
# **Постановка задачи**

Наша задача состоит в классификации типа пейзажа. Всего представлено $k = 6$ видов пейзажей: постройки, леса, горы, ледники, улицы и моря. Благодаря классификации кадров мы получим возможность классифицировать и понимать местность на фото, что значительно улучшит пользовательский опыт, упростив поиск нужного фильма простым использованием фильтра камеры телефона.

## Формат входных и выходных данных

Входные данные представляют собой изображения в формате JPEG. В качестве меток классификации предлагаются названия сцен, которые показаны на изображении.

## Метрики

В качестве основной метрики предлагается использовать точность. Точность классификации должна составлять не менее 90%. Для более точной оценки качества классификации возможно использование метрики F1.

## Валидация

У нас уже имеется готовый датасет, который размечен и разделён на две выборки. Чтобы убедиться, что датасет будет сбалансирован, мы визуализируем его и оценим соотношения. Если датасет окажется несбалансированным, и этот аспект покажется нам критически важным, мы перераспределим изображения, чтобы сделать датасет более сбалансированным.

## Данные

[Ссылка на датасет.](https://www.kaggle.com/datasets/nitishabharathi/scene-classification)

Все объекты датасета - изображения в формате .jpg, поэтому обрабатывать их предполагается модулем Image из библиотеки PIL, что имеет методы для открытия изображений и представления их в формате массива 3xHxW.

# **Моделирование**

## Бейзлайн

Наша задача представляет собой многоклассовое предсказание поэтому в качестве бейзлайна лучше всего подойдёт метод присвоения самого распространённого класса.

## Основная модель

В качестве основной модели предполагается использовать [одну из архитектур resnet](https://pytorch.org/hub/pytorch_vision_resnet), для оптимизации параметров подойдёт Adam, для вычисления потерь подойдёт мультиклассовая кросс-энтропия.

# **Внедрение**

Предполагается создать для нашей модели веб-приложение, которое возможно запустить на локальном устройстве. В качестве дополнительной функции можно добавить возможность загружать видео, которое будет разбиваться на кадры и выдавать массив меток и одну метку, которая классифицирует контекст самой видеозаписи.
